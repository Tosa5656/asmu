smlog

Утилита и API для чтения, поиска и мониторинга системных логов (файлы и systemd journal).

Назначение

Чтение и мониторинг логов, в том числе journald. Поиск по ключевым словам, топ IP и пользователей, готовые отчёты (безопасность, система), мониторинг в реальном времени с правилами.

Исходники

  • smlog/smlog.cpp — точка входа и команды (list, read, search, journal, top-ips, report, monitor).
  • smlog/SystemLogger.cpp, SystemLogger.h — работа с файловыми логами и journal.

Команды CLI

КомандаОписание
smlog helpСправка
smlog listДоступные логи
smlog read <файл> [N]Чтение последних N строк (например /var/log/syslog 100)
smlog search "строка" <файл>Поиск по ключевым словам
smlog journal <unit> [N]Журнал systemd (например sshd 50)
smlog top-ips <файл> [N]Топ IP по логу
smlog report securityОтчёт по безопасности
smlog monitorМониторинг в реальном времени (Ctrl+C — выход)

API (smlog_api.h)

Класс SecurityManager::LogAnalyzer. Типы: LogEntry, LogFilter. Результаты — LogResult<T> с кодами LogError.

Методы: чтение файла логов, поиск по файлу, работа с journal, отчёты. Подробности в заголовке api/include/smlog_api.h.
