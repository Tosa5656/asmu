Архитектура

Структура репозитория и связи между компонентами.

Дерево каталогов

asmu/
├── api/                    # C++ API
│   ├── include/            # Заголовки (asmu.h, sm*_api.h)
│   └── src/                # Реализация API
├── argsparser/             # Парсер аргументов командной строки
├── doc/                    # Документация (Doxygen, attacks)
│   └── attacks/            # HTML-описания атак MITRE ATT&CK
├── docs/                   # Документация для разработчиков (этот сайт)
├── json/                   # nlohmann/json для работы с JSON
├── logger/                 # Потокобезопасный логгер
├── smdb/                   # Утилита и логика базы атак
├── smlog/                  # Утилита и логика логов (SystemLogger)
├── smnet/                  # Утилита и логика сети (portScanner, trafficMonitor)
├── smpass/                 # Утилита и хранилище паролей (AES256, SHA256)
├── smssh/                  # Утилита SSH (sshConfig, sshAttackDetector)
└── test/                   # Тесты и тестовые данные

Поток данных

  • Утилиты (smssh, smlog, smpass, smnet, smdb) — точки входа: парсят аргументы (argsparser), вызывают логику модуля, пишут в лог (logger) и выводят результат в stdout/stderr.
  • API — те же возможности доступны через классы в namespace Asmu; реализация в api/src вызывает код из smssh, smlog и т.д. или дублирует логику там, где это необходимо.
  • smdb читает HTML-файлы из doc/attacks/ и по ним строит поиск и выдачу рекомендаций; при установке документация копируется в /usr/share/doc/asmu/.
